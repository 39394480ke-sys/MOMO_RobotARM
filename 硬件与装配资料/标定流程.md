# 标定流程

标定是把真实舵机 raw 位置和软件中的逻辑关节角度对齐。连接不是标定。

## 基本原则

- `connect()` 每次只读取已有标定文件，不会重新标定。
- 连接不会要求用户移动机械臂。
- 连接不会覆盖 `home_present_raw`、`zero_present_raw`、`range_min`、`range_max`。
- 只有零点、限位、舵机 ID、装配姿态或减速结构变化时，才重新标定。
- 默认先 dry-run；真实移动必须先通过安全检查。

## 标定程序

如果项目采用阶段编号目录命名，命令为：

```bash
python ../04_真实舵机控制/标定程序_calibrate.py
```

当前仓库实际目录名为 `真实舵机控制`，可在 `硬件与装配资料` 目录执行：

```bash
python ../真实舵机控制/标定程序_calibrate.py
```

应用已有标定，不重新生成标定文件：

```bash
python ../04_真实舵机控制/标定应用_apply_calibration.py
```

当前仓库实际命令：

```bash
python ../真实舵机控制/标定应用_apply_calibration.py
```

`标定应用_apply_calibration.py` 只把已有标定文件中的寄存器配置写入舵机，不重新读取 `home_present_raw`，不重新计算 `zero_present_raw`，不重新记录 `range_min/range_max`，不修改标定文件内容。

## 标定内容

### J1 / J4 / 夹爪：单圈限位标定

涉及关节：

```text
J1 shoulder_pan 底座旋转
J4 wrist_flex 腕部俯仰
J6 gripper 夹爪
```

需要记录：

- `zero_present_raw`：单圈关节逻辑 0 度对应的舵机 raw。
- `range_min`：该单圈关节允许的最小 raw。
- `range_max`：该单圈关节允许的最大 raw。

### J2 / J3 / J5：多圈 absolute raw home 标定

涉及关节：

```text
J2 shoulder_lift 肩部抬升
J3 elbow_flex 肘部弯曲
J5 wrist_roll 腕部旋转
```

需要记录：

- `home_present_raw`：多圈关节逻辑 0 度对应的舵机 raw。
- `phase=28`：多圈 absolute raw 配置。

多圈关节不做 4096 包裹，目标 raw 使用 absolute raw，并受 `MULTI_TURN_ABSOLUTE_RAW_LIMIT = 30719` 保护。

## 字段解释

| 字段 | 含义 |
|---|---|
| `zero_present_raw` | 单圈关节逻辑 0 度对应的舵机 raw |
| `home_present_raw` | 多圈关节逻辑 0 度对应的舵机 raw |
| `range_min` | 单圈关节安全范围最小 raw |
| `range_max` | 单圈关节安全范围最大 raw |
| `phase=28` | 多圈 absolute raw 配置 |
| `RAW_COUNTS_PER_REV=4096` | 舵机每圈 raw 计数 |
| `POSITION_MODE_VALUE=0` | 位置模式值 |

## 标定后检查

1. 查看 `真实舵机控制/标定文件.json` 是否更新。
2. 运行阶段十一标定检查：

```bash
cd 系统集成
python 标定检查.py
```

3. 先 dry-run，再真实小角度单关节测试。
