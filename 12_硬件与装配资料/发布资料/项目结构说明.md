# 项目结构说明

当前仓库没有重命名为 `00_...` 到 `12_...` 的全部数字目录。为了不推翻已有项目，下面用“阶段编号 / 当前目录名”对应说明。

| 阶段目录 | 当前实际目录 | 负责什么 | 输入是什么 | 输出是什么 | 依赖谁 | 不负责什么 |
|---|---|---|---|---|---|---|
| `00_学习日志` | 待补或用 `12_硬件与装配资料/学习日志模板.md` | 记录学习过程、问题和测试结果 | 每日开发记录 | 学习日志 | 全项目 | 不控制机械臂 |
| `01_项目总览` | `项目整体结构.md` / 发布资料 | 说明项目目标和模块边界 | 阶段文档 | 总览说明 | 全项目 | 不写具体控制代码 |
| `02_机械臂基础概念` | `机械臂基本概念` | 解释关节、安全、名词 | 原理学习 | 基础概念文档 | 无 | 不连接真实舵机 |
| `03_仿真控制系统` | `仿真控制系统` | 软件仿真、姿态和动作基础 | 配置、动作文件 | 仿真姿态和动作结果 | 基础概念 | 不写真实舵机 |
| `04_真实舵机控制` | `真实舵机控制` | Feetech 舵机、标定、dry-run、真实控制 | 串口、标定文件、目标角度 | 舵机状态或移动结果 | 基础概念、仿真逻辑 | 不做 Web/GUI/视觉 |
| `05_URDF运动学仿真` | `URDF运动学仿真` | URDF、FK、IK、3D 显示 | URDF、mesh、目标位姿 | 关节解、3D 显示 | 阶段四关节定义 | 不直接绕过阶段四写舵机 |
| `06_动作录制与回放增强` | `动作录制与回放增强` | 录制、插值、回放、安全摘要 | 姿态序列、动作文件 | 动作库和回放结果 | 阶段三、四、五 | 不做硬件装配 |
| `07_GUI图形界面` | `GUI图形界面` | 桌面图形控制界面 | 控制器状态、用户输入 | GUI 操作结果 | 阶段四、五、六 | 不直接修改标定策略 |
| `08_Web控制台` | `Web控制台` | Web API、前端、Quick Control | HTTP 请求、动作命令 | API 响应和状态 | 阶段四、六 | 不让外部绕过安全检查 |
| `09_视觉识别与跟随` | `视觉识别与跟随` | 摄像头、检测、偏移计算、跟随 | 摄像头画面、Web API | 跟随命令或 dry-run 结果 | 阶段八 | 不负责机械装配 |
| `10_语音Agent` | `语音Agent` | 语音输入、STT/TTS、Agent 工具桥接 | 用户语音或文本 | 工具调用和回复 | 阶段八 | 不直接写 raw |
| `11_系统集成` | `系统集成` | 一键启动、健康检查、统一配置和日志 | 总配置、环境变量 | 系统运行状态 | 阶段四到十 | 不替代单阶段调试 |
| `12_硬件与装配资料` | `12_硬件与装配资料` | BOM、装配、接线、标定、安全、发布资料 | 原硬件逻辑、现有代码和文档 | 可复刻资料包 | 全项目 | 不继续新增控制算法 |

## 目录边界原则

- 硬件参数以阶段四和原 MomoAgent / SOARM MOCE 逻辑为准。
- URDF mesh 对应以阶段五当前文件为准。
- Web、GUI、视觉、Agent 不应绕过阶段四安全检查。
- 真实模式只能在标定完整、dry-run 通过、安全检查完成后使用。
