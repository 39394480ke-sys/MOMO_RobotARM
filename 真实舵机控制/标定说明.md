# 标定说明

标定不是驱动库。`lerobot / feetech-servo-sdk` 是通信依赖，`标定程序_calibrate.py` 是使用这些依赖连接真实舵机、读取/写入寄存器并生成 `标定文件.json` 的程序。

## 三种模式区别

dry-run：不需要真实依赖，不连接硬件，只检查角度到 raw 的映射。

标定程序：需要真实依赖，需要连接硬件，会读取当前位置，必要时写入寄存器并生成 `标定文件.json`。

真实控制：需要真实依赖，需要连接硬件。`connect()` 每次只读取 `标定文件.json`，不会重新标定，也不会覆盖 `home_present_raw / zero_present_raw`。

## 什么时候需要重新标定

- 更换舵机。
- 舵机 ID 或方向变了。
- 机械臂结构拆装后零点变了。
- 单圈关节安全范围变了。
- 多圈关节的逻辑 0 度位置变了。
- `状态` 显示的逻辑角度明显不符合真实姿态。

运行：

```bash
mamba activate momo_rebot
cd 真实舵机控制
python 标定程序_calibrate.py
```

## 什么时候只需要应用已有标定

- 舵机寄存器被清空。
- 换电脑后想恢复舵机寄存器配置。
- 只想重新写入 `Phase / Homing_Offset / Limit`，不想重新完整标定。

运行：

```bash
mamba activate momo_rebot
cd 真实舵机控制
python 标定应用_apply_calibration.py
```

## 多圈关节 J10/J11/J12/J13/J15

多圈关节固定为：

```text
j10 = J10_底盘导轨
j11 = J11_底座旋转（1:5 行星减速）
j12 = J12_肩部抬升
j13 = J13_肘部弯曲
j15 = J15_腕部旋转
```

标定逻辑：

- `Operating_Mode = 0`
- `Homing_Offset = 0`
- `Phase = 28`
- `Min_Position_Limit = 0`
- `Max_Position_Limit = 0`
- 当前 `Present_Position` 记录为 `home_present_raw`
- 当前 `Present_Position % 4096` 记录为 `home_present_wrapped_raw`

多圈关节不使用普通 `range_min / range_max`，标定文件中固定为 `0/0`。
后续 `home()` 回到的是相对 `home_present_raw` 的 0 度。

J10 底盘导轨和 J11 减速底盘旋转不需要手动推动来采样限位。
运行标定程序时，请先把导轨和底盘停在安全位置；程序会直接把当前 J10/J11 `Present_Position` 记录为对应的逻辑 0 / Home。

## 单圈关节 J14/夹爪

单圈有限位标定关节：

```text
j14 = J14_腕部俯仰
gripper = 夹爪_ID16
```

如果已有完整单圈标定，标定程序会询问是否复用。按 ENTER 复用，只更新 J10/J11/J12/J13/J15 的多圈 home。输入 `c` 才重新标定 J14/夹爪。

J11 已改为 1:5 减速多圈底盘旋转，不再手动采样 `range_min / range_max`。旧 `标定文件.json` 中的 J11 单圈记录已过期，真实移动前请重新运行标定程序生成 J11 多圈 `home_present_raw`。

标定程序启动时不会再要求输入“我确认开始标定”。
如果需要重新标定单圈关节，仍然会在单圈流程里要求你按 ENTER 确认姿态和采样开始 / 结束。

重新标定单圈关节时，先把 J14 和夹爪摆到希望的零点姿势。
你按 ENTER 后，程序会调用 `set_half_turn_homings()`，把这些关节的当前姿势写到 raw 中点附近（2047/2048）。然后再开始记录 `range_min / range_max`。这样采样范围通常会围绕中点展开，避免像 `21..4091` 这种跨过 0 的严重跳变区间。

重新标定时：

- 程序先关闭扭矩。
- 手动把 J14 放到希望的 0 度位置。
- 手动把夹爪放到安全位置。
- J14/夹爪统一调用 `set_half_turn_homings()`，把当前零点姿势居中到 2047/2048 附近。
- 记录 J14/夹爪的 `range_min / range_max`。

## 第一次真机前安全流程

1. 确认机械臂断开负载或处于安全位置。
2. 确认电源稳定。
3. 确认串口正确。
4. 保持 `dry_run: true`。
5. 运行读取状态脚本。
6. 运行多圈映射检查。
7. 确认标定文件内容。
8. 必要时运行 `标定应用_apply_calibration.py`。
9. 必要时运行 `标定程序_calibrate.py`。
10. 真机第一次移动只测试一个关节。
11. 单次角度不超过 2-5 度。
12. 随时准备断电。
