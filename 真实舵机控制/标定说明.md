# 标定说明

标定不是驱动库。`lerobot / feetech-servo-sdk` 是通信依赖，`标定程序_calibrate.py` 是使用这些依赖连接真实舵机、读取/写入寄存器并生成 `标定文件.json` 的程序。

## 三种模式区别

dry-run：不需要真实依赖，不连接硬件，只检查角度到 raw 的映射。

标定程序：需要真实依赖，需要连接硬件，会读取当前位置，必要时写入寄存器并生成 `标定文件.json`。

真实控制：需要真实依赖，需要连接硬件。`connect()` 每次只读取 `标定文件.json`，不会重新标定，也不会覆盖 `home_present_raw / zero_present_raw`。

## 什么时候需要重新标定

- 更换舵机。
- 舵机 ID 或方向变了。
- 机械臂结构拆装后零点变了。
- 单圈关节安全范围变了。
- 多圈关节的逻辑 0 度位置变了。
- `状态` 显示的逻辑角度明显不符合真实姿态。

运行：

```bash
mamba activate momo_rebot
cd 真实舵机控制
python 标定程序_calibrate.py
```

## 什么时候只需要应用已有标定

- 舵机寄存器被清空。
- 换电脑后想恢复舵机寄存器配置。
- 只想重新写入 `Phase / Homing_Offset / Limit`，不想重新完整标定。

运行：

```bash
mamba activate momo_rebot
cd 真实舵机控制
python 标定应用_apply_calibration.py
```

## 多圈关节 J2/J3/J5

多圈关节固定为：

```text
shoulder_lift = J2_肩部抬升
elbow_flex = J3_肘部弯曲
wrist_roll = J5_腕部旋转
```

标定逻辑：

- `Operating_Mode = 0`
- `Homing_Offset = 0`
- `Phase = 28`
- `Min_Position_Limit = 0`
- `Max_Position_Limit = 0`
- 当前 `Present_Position` 记录为 `home_present_raw`
- 当前 `Present_Position % 4096` 记录为 `home_present_wrapped_raw`

多圈关节不使用普通 `range_min / range_max`，标定文件中固定为 `0/0`。  
后续 `home()` 回到的是相对 `home_present_raw` 的 0 度。

## 单圈关节 J1/J4/夹爪

单圈有限位标定关节：

```text
shoulder_pan = J1_底座旋转
wrist_flex = J4_腕部俯仰
gripper = 夹爪_ID6
```

如果已有完整单圈标定，标定程序会询问是否复用。按 ENTER 复用，只更新 J2/J3/J5 的多圈 home。输入 `c` 才重新标定 J1/J4/夹爪。

重新标定时：

- 程序先关闭扭矩。
- 手动把 J1 放到安全中间位置。
- 手动把 J4 放到希望的 0 度位置。
- 手动把夹爪放到安全位置。
- J1 尽量调用 `set_half_turn_homings(["shoulder_pan"])`。
- J4 不强制 half-turn homing，优先保留旧 `homing_offset`。
- gripper 的 `Homing_Offset` 固定为 `2047`。
- 记录 J1/J4/夹爪的 `range_min / range_max`。

## 第一次真机前安全流程

1. 确认机械臂断开负载或处于安全位置。
2. 确认电源稳定。
3. 确认串口正确。
4. 保持 `dry_run: true`。
5. 运行读取状态脚本。
6. 运行多圈映射检查。
7. 确认标定文件内容。
8. 必要时运行 `标定应用_apply_calibration.py`。
9. 必要时运行 `标定程序_calibrate.py`。
10. 真机第一次移动只测试一个关节。
11. 单次角度不超过 2-5 度。
12. 随时准备断电。
