# 故障排查手册

每个问题按“现象、可能原因、检查方法、解决办法、相关文件”记录。排查真实硬件时，优先断开动作入口，保留急停或断电手段。

## 1. 程序启动失败

现象：
程序退出、找不到文件、参数报错。

可能原因：
当前目录不对，命令路径不对，配置文件缺失。

检查方法：
确认 `pwd`，查看 README 中的运行目录。

解决办法：
切换到对应阶段目录再运行；先运行阶段十一依赖检查。

相关文件：
`系统集成/依赖检查.py`，`系统集成/README_系统集成.md`

## 2. Python 版本错误

现象：
语法报错、类型标注报错、依赖安装失败。

可能原因：
没有使用项目环境，Python 版本过低。

检查方法：
运行 `python --version`。

解决办法：
进入项目使用的 Python 环境，再重新安装依赖。

相关文件：
各阶段 README，`真实舵机控制/依赖安装说明.md`

## 3. 依赖缺失

现象：
`ModuleNotFoundError` 或 `ImportError`。

可能原因：
没有安装 `pyserial`、`pyyaml`、GUI/Web/视觉/语音相关依赖。

检查方法：
运行 `cd 系统集成 && python 依赖检查.py`。

解决办法：
按缺失模块安装；ARM 开发板真实舵机默认走轻量 SDK，重点确认 `feetech-servo-sdk pyserial scservo_sdk pyyaml`。

相关文件：
`系统集成/依赖检查.py`，`真实舵机控制/标定工具_calibration_utils.py`

## 4. 串口找不到

现象：
打开串口失败，找不到 Mac 的 `/dev/tty.usbmodemXXXX` 或开发板的 `/dev/momo-servo`。

可能原因：
USB 未连接，线材不是数据线，串口名变化，权限问题，或开发板 CH343 驱动没有接管 `1a86:55d3`。

检查方法：
查看系统串口列表，开发板运行 `lsusb -t` 确认 `Driver=usb_ch343`，并检查 `.env` 中的 `ARM_ROBOT_PORT`。

解决办法：
更换 USB 线，重新插拔，开发板执行 `bash scripts/setup_qinheng_55d3_serial.sh`，并设置 `ARM_ROBOT_PORT=/dev/momo-servo`。

相关文件：
`真实舵机控制/真实配置.yaml`，`发布资料/.env.example`

## 5. 舵机无响应

现象：
串口存在但舵机不动或读不到状态。

可能原因：
舵机电源未接，通信线松动，ID 不对，总线方向接反。

检查方法：
检查电源指示和接线，运行读取舵机状态脚本。

解决办法：
断电后重插线，确认电源和 ID，先读取状态再移动。

相关文件：
`真实舵机控制/测试脚本_test/01_读取舵机状态.py`

## 6. 舵机 ID 不对

现象：
控制 J2 时另一个关节动，或某个 ID 读不到。

可能原因：
舵机 ID 写错或装配位置与 ID 不一致。

检查方法：
逐个读取 ID，对照 `舵机ID与关节对应表.md`。

解决办法：
修正舵机 ID 或重新安装到正确位置，不要改 joint_scales 来掩盖 ID 错误。

相关文件：
`舵机ID与关节对应表.md`

## 7. 标定文件缺失

现象：
真实模式禁止移动，提示标定不完整。

可能原因：
`标定文件.json` 不存在或字段缺失。

检查方法：
运行 `cd 系统集成 && python 标定检查.py`。

解决办法：
必要时运行阶段四标定程序；已有标定只需应用时运行应用标定。

相关文件：
`真实舵机控制/标定文件.json`，`标定流程.md`

## 8. 关节方向反了

现象：
逻辑正角度和实物运动方向相反。

可能原因：
装配方向、joint_scale 或标定方向不一致。

检查方法：
先 dry-run，再真实小于 2 度单关节测试。

解决办法：
不要随意改固定 joint_scales；先确认装配、ID、标定字段。

相关文件：
`真实舵机控制/角度映射_angle_mapper.py`，`真实舵机控制/真实配置.yaml`

## 9. 多圈关节跑飞

现象：
J2/J3/J5 运动幅度异常或越转越远。

可能原因：
`home_present_raw` 错误，phase 不是 28，把多圈当单圈包裹。

检查方法：
检查标定文件中 J2/J3/J5 的 `home_present_raw` 和 `phase`。

解决办法：
停止真实移动，重新确认 home 姿态，必要时重新标定多圈关节。

相关文件：
`标定流程.md`，`真实舵机控制/角度映射_angle_mapper.py`

## 10. 夹爪不动

现象：
夹爪没有开合，或动作范围很小。

可能原因：
ID 6 未连接，夹爪 range_min/range_max 不对，机械卡滞。

检查方法：
断电手动检查夹爪，读取 ID 6 状态。

解决办法：
修复机械卡滞，重新确认夹爪限位标定。

相关文件：
`真实舵机控制/标定文件.json`

## 11. URDF 显示不对

现象：
3D 模型错位、mesh 丢失、关节方向不对。

可能原因：
mesh 路径错误，URDF joint/link 对应关系不清，模型偏移未调整。

检查方法：
运行 URDF 检查脚本和 3D 仿真显示测试。

解决办法：
检查 `STEP_STL_URDF对应关系.md` 和 URDF mesh 引用。

相关文件：
`URDF运动学仿真/urdf/soarmoce_urdf.urdf`

## 12. IK 无解

现象：
目标位姿求解失败或结果超限。

可能原因：
目标超出工作空间，关节限位过窄，初值不合适。

检查方法：
先运行正运动学测试，再运行逆运动学测试。

解决办法：
换小一点的末端增量，确认目标在可达范围内。

相关文件：
`URDF运动学仿真/逆运动学_ik.py`

## 13. Web API 启动失败

现象：
Web 服务无法启动或端口被占用。

可能原因：
依赖缺失，端口冲突，配置路径错误。

检查方法：
运行 `cd Web控制台 && python 测试脚本_test/02_health测试.py`。

解决办法：
更换端口或停止旧服务，按 README 安装依赖。

相关文件：
`Web控制台/README_Web控制台.md`

## 14. GUI 启动失败

现象：
GUI 窗口打不开或页面报错。

可能原因：
Qt 依赖缺失，运行目录错误，配置文件缺失。

检查方法：
运行 `cd GUI图形界面 && python 测试脚本_test/01_启动GUI.py`。

解决办法：
安装 GUI 依赖，确认 `GUI配置.yaml` 存在。

相关文件：
`GUI图形界面/README_GUI图形界面.md`

## 15. 摄像头打不开

现象：
摄像头预览失败或没有画面。

可能原因：
摄像头被占用，设备索引错误，权限未授权。

检查方法：
运行 `cd 视觉识别与跟随 && python 测试脚本_test/01_摄像头预览.py`。

解决办法：
关闭占用摄像头的软件，修改摄像头索引，检查 USB 连接。

相关文件：
`视觉识别与跟随/视觉配置.yaml`

## 16. STT/TTS 不可用

现象：
语音识别或语音播报失败。

可能原因：
环境变量缺失，服务地址错误，API Key 未配置。

检查方法：
运行语音 Agent 的 STT/TTS 测试脚本。

解决办法：
按 `发布资料/.env.example` 配置服务；不要提交真实密钥。

相关文件：
`语音Agent/README_语音Agent.md`，`发布资料/.env.example`

## 17. Agent 工具调用失败

现象：
Agent 能回复文本，但不能控制机械臂或工具报错。

可能原因：
Web API 未启动，工具桥接配置错误，安全策略拦截。

检查方法：
先测试 Web API，再运行工具桥接 dry-run 测试。

解决办法：
保持 Agent 通过阶段八 API 间接控制，不让 Agent 直接写 raw。

相关文件：
`语音Agent/agent/工具桥接_tool_bridge.py`，`语音Agent/agent/安全策略_safety_policy.py`

故障记录 CSV 见 `表格/故障记录.csv`。
